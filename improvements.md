# Pontos de melhoria do projeto

## 1. Testes
- Adicionar testes unitários e de integração.
- Utilizar Vitest + React Testing Library para validar componentes e hooks.

## 2. Tratamento de erro
- Melhorar o tratamento de erros nas requisições de API.
- Padronizar estados de carregamento, falha e vazio em todas as telas.

## 3. Organização do código
- Padronizar melhor a lógica entre os módulos de animes e mangás.
- Centralizar regras reutilizáveis em hooks e serviços compartilhados.

## 4. Performance
- Otimizar carregamento de imagens e listas longas.
- Avaliar uso de prefetch de rotas e cache mais estratégico.

## 5. Acessibilidade
- Revisar foco, contraste, labels e navegação por teclado.
- Melhorar semântica de componentes e interações.

## 6. Experiência do usuário
- Tornar busca, filtros e navegação mais consistentes entre páginas.
- Adicionar mais feedback visual para ações como salvar em watchlist.

## 7. Manutenibilidade
- Documentar convenções de desenvolvimento e estrutura de pastas.
- Criar padrões para commits, versionamento e contribuição.

## Prioridade recomendada
1. Testes
2. Tratamento de erro e estados vazios
3. Acessibilidade
4. Padronização da arquitetura e hooks
